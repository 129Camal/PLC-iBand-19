Abstract:

Decidimos utilizar BootStrap para fazer as views em Pug. Possuimos também quatro coleções na base de dados em MongoDB, (users, works, calendar e news) cada uma desta tem os seus modelos, controladores e consequentemente rotas API para ir buscar a meta-informação à base de dados.

Temos 3 tipos de utilizador (consumidor, produtor e administrador) em que temos uma rota para cada, fazendo autenticação usando o Passport. Para previnir que utilizadores que efetuaram o login acedam a rotas que não lhe pertencem verificamos sempre qual o tipo de utilizador logado que está a fazer o pedido no req.user.

O consumidor apenas pode visualizar os conteúdos, já o produtor pode adicionar/remover conteúdo e com isto as operações consequentes são efetuadas à base de dados e ao file system.

Já o administrador pode efetuar todas as operações do produtor e para além disso, tem a possibilidade de gerir os utilizadores(adicionar, remover, alterar) e ver estatisticas de utilização da aplicação como, número de visualizações de cada obra e número de downloads das partituras de cada obra.

Para além disso, nas adições das obras por parte do produtor/administrador utilizamos JQuery para aliviar carga do servidor, permitindo assim ao utilizador criar mais campos para adicionar instrumentos relativos à obra em inserção.

Por fim, também utilizamos o Winston para guardar os logs num ficheiro de texto para assim ter acesso a qualquer momento.

Para carregar os ficheiros das obras para a base de dados, tem que se utilizar a função presente na promessa na ligação à base de dados, para colocar os documentos na base de dados de maneira a criar _id para cada instrumento.
